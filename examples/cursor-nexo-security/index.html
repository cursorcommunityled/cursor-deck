Capa 01 · Rules
01  Antes

Cursor Rules.

Se pegan una vez. Cursor las respeta siempre.

01
Crea la carpeta
.cursor/rules/
02
Adentro, crea el archivo
security.mdc
03
Pega las reglas
→
security.mdc
---
description: Security baseline
alwaysApply: true
---

# Auth
- Valida la sesión del servidor,
  NUNCA confíes en el cliente.
- user.id == session.user.id
- Secretos: process.env, jamás
  hardcodeados.
- JWT: exp <= 15m, alg explícito.

# Prompt injection
- NUNCA interpolés input de
  usuario en prompts del sistema.
- Separa role:system / role:user.

Capa 02 · Prompt
02  Durante

Un prompt que puedes copiar.

Pegalo antes de pedir features sensibles: login · pagos · uploads · auth.

Antes de escribir el código:
1.
Lista los 5 riesgos de seguridad más comunes para este tipo de feature.
2.
Muéstrame cómo los mitigas (librería, validación, config).
3.
Solo entonces, escribe el código.

Si tienes Rules, agrega « …siguiendo security.mdc » al final.

backend/src/auth/auth.service.ts
async login(username: any, password: any) {
  // Input va directo a Mongo — NoSQL injection
  return this.db.collection('users')
    .findOne({ username, password });
}

async register(username: any, password: any) {
  // Password en texto plano, sin bcrypt
  await this.db.collection('users')
    .insertOne({ username, password });
}
